# Используемые коллекции

В проекте используются следующие коллекции:

- `users` — пользователи системы
- `posts` — публикации пользователей
- `messages` — личные сообщения

  

# Коллекция users

Коллекция хранит информацию о пользователях.

Пример структуры документа:

```json
{
  "_id": "ObjectId",
  "login": "string",
  "email": "string",
  "password": "string",
  "first_name": "string",
  "last_name": "string",
  "age": "number",
  "interests": ["string"],
  "created_at": "date"
}
````

## Описание полей

* `_id` — уникальный идентификатор пользователя
* `login` — логин
* `email` — электронная почта
* `password` — пароль
* `first_name` — имя
* `last_name` — фамилия
* `age` — возраст
* `interests` — список интересов
* `created_at` — дата регистрации

  

# Коллекция posts

Коллекция хранит посты пользователей.

Пример структуры документа:

```json
{
  "_id": "ObjectId",
  "author_id": "ObjectId",
  "content": "string",
  "tags": ["string"],
  "likes": ["ObjectId"],
  "comments": [
    {
      "user_id": "ObjectId",
      "text": "string",
      "created_at": "date"
    }
  ],
  "created_at": "date"
}
```

## Описание полей

* `_id` — идентификатор поста
* `author_id` — ссылка на автора поста
* `content` — текст публикации
* `tags` — теги поста
* `likes` — массив пользователей, поставивших лайк
* `comments` — комментарии к посту
* `created_at` — дата создания поста

  

# Коллекция messages

Коллекция хранит личные сообщения между пользователями.

Пример структуры документа:

```json
{
  "_id": "ObjectId",
  "from_user_id": "ObjectId",
  "to_user_id": "ObjectId",
  "text": "string",
  "is_read": "boolean",
  "created_at": "date"
}
```

## Описание полей

* `_id` — идентификатор сообщения
* `from_user_id` — отправитель
* `to_user_id` — получатель
* `text` — текст сообщения
* `is_read` — статус прочтения
* `created_at` — дата отправки

  

# Embedded documents и References

В MongoDB можно хранить связанные данные двумя способами:

* embedded documents
* references

В проекте использовались оба варианта.

## Embedded documents

Комментарии (`comments`) реализованы как embedded documents внутри поста.

Пример:

```json
{
  "comments": [
    {
      "user_id": "ObjectId",
      "text": "Nice post",
      "created_at": "date"
    }
  ]
}
```

### embedded

Комментарии напрямую относятся к конкретному посту и обычно запрашиваются вместе с ним.
Такой подход уменьшает количество отдельных запросов к базе данных.

  

## References

Для связей между пользователями, постами и сообщениями используются references.

Пример:

```json
{
  "author_id": "ObjectId"
}
```

или

```json
{
  "from_user_id": "ObjectId",
  "to_user_id": "ObjectId"
}
```

### references

Пользователь является отдельной сущностью и может использоваться во многих документах одновременно.

Использование references позволяет:

* избежать дублирования данных
* уменьшить размер документов
* проще обновлять данные пользователей

  

# Использование массивов

В проекте активно используются массивы MongoDB.

## Пример tags

```json
"tags": ["mongodb", "backend"]
```

## Пример likes

```json
"likes": ["ObjectId", "ObjectId"]
```

## Пример interests

```json
"interests": ["music", "sport"]
```

  

# Используемые типы данных MongoDB

В проекте используются разные типы данных MongoDB:

| Тип      | Где используется          |
|     -- |                 - |
| String   | login, content, text      |
| Number   | age                       |
| Boolean  | is_read                   |
| Date     | created_at                |
| Array    | likes, tags, interests    |
| Object   | comments                  |
| ObjectId | идентификаторы документов |

  

# Общая логика структуры

Основная логика модели заключается в следующем:

* пользователи хранятся отдельно
* посты связаны с пользователями через references
* сообщения также связаны через references
* комментарии встроены в посты как embedded documents

Такой подход позволяет сделать структуру более гибкой и уменьшить количество лишних запросов.
